# Core Nano Banana Finished Artwork Design

## Objective

Replace the Core programmatic SVG composition with two professionally art-directed, fully rasterized Nano Banana Pro images. The generated images own the complete composition, typography, diagrams, and Capell branding.

![Current Core artwork to replace](../../../packages/core/docs/assets/readme/hero.jpg)

## Brand Input

The canonical Capell wordmark is supplied to Nano Banana as an image reference, never described only in text. The workflow rasterizes `artwork/foundation-series/capell-logo.svg` to a high-resolution transparent PNG and passes that PNG through Nano Banana's input-image option.

Every prompt must instruct Nano Banana to preserve the supplied wordmark exactly: curved A, broken underline/separator, proportions, navy colour, clear space, and original letterforms. A candidate is rejected if the logo is redrawn, misspelled, distorted, or replaced with generic typography.

## Finished Outputs

Nano Banana generates two independent finished compositions:

- `packages/core/docs/assets/readme/hero.jpg` at 2880×960;
- `packages/core/docs/assets/marketplace/extension-card.jpg` at 800×500.

The marketplace card is not a crop of the hero. It uses a simpler hierarchy designed to remain clear at 400×250.

Both images retain the approved structural-spine product story: Capell Core lives inside a Laravel application, Admin has a controlled relationship with Core, extension seams are explicit, and the application-owned Frontend can produce distinctive sites from shared structure. Nano Banana may express those ideas through a richer editorial architectural composition instead of literal SVG boxes and rails.

## Production Workflow

1. Rasterize the canonical Capell logo to a transparent reference PNG.
2. Generate multiple meaningful 4K hero candidates with Nano Banana using the logo reference.
3. Review candidates for brand fidelity, product meaning, professional art direction, hierarchy, and absence of generic factory/machinery imagery.
4. Generate the marketplace card independently from the approved direction, again using the logo reference.
5. Iterate both assets until the hero, card, and 400×250 reduction have no important visual or product-story issues.
6. Resize and encode the approved outputs as progressive sRGB JPEGs at exact dimensions.

ImageMagick may be used only for logo rasterization, resizing, colour-space normalization, JPEG encoding, and inspection. It must not compose the artwork, add text, draw diagrams, or act as a replacement visual generator.

## Repository Simplification

Remove the Core SVG composition pipeline:

- delete `artwork/foundation-series/render-core.js`;
- remove the Core renderer invocation from `artwork/foundation-series/render.sh`;
- remove renderer-specific background and composition assets that are no longer used;
- rewrite `artwork/foundation-series/README.md` around the Nano Banana reference-image workflow;
- retain a lightweight `verify-core.js` validator for paths, exact dimensions, progressive sRGB encoding, file-size limits, and review evidence.

The canonical Capell SVG may remain only as the logo reference source for Nano Banana. It is not used to build a finished composition.

## Acceptance

- The complete visual composition is generated by Nano Banana Pro.
- The real Capell logo is supplied as a reference image and remains faithful.
- No SVG/HTML/canvas/programmatic layout generator produces either finished asset.
- Hero and marketplace card are separately art-directed.
- The card remains strong at 400×250.
- Outputs are exact progressive sRGB JPEGs at their required dimensions.
- Existing README, manifest, and marketplace paths remain unchanged.
- The artwork contract, documentation screenshot gate, and focused Core manifest/marketplace tests pass.
- Unrelated dirty-tree work is preserved and excluded from the artwork commit.
